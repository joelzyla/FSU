Week 1 Research

Data Research and Project Intent

Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

Using twitter data could be helpful on our mobile applications for logins and payments.

By using "log in with twitter", our users don't have to go through a long signup process when making an account. Another benefit is the user won't have to remember another password. We can also pull information like a profile picture, which will make our user information more complete.

The twitter API can also be used when making a payment. By automatically pulling a users name and address, we have sped up the checkout process, which makes it easier for the user and will probably decrease the amount of abandanded carts.

Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

By choosing JSON-P, we get padding, which makes our life easier. JSON is already used in javascript, and by using JSON-P we won't need to worry about cross domain issues.

Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

There are several data resources that would pair well with native device features. Weather data API's could be paired with location to get your local weather. Twitter data could be paired with notifications to be alerted about new posts. Photography data from Flikr could be paired with the camera to add new pictures, or with local storage to save pictures from Flikr to your device.

Please describe any design and functionality intentions you may have for your Demo app components this month.

Because we are using the iPad, I am considering trying to implement a split-view layout(using a jqm plugin). I also like the idea of having only a top header(no footer), and having menu items drop down when you click a button in the header. This allows maximum use of the screen for other purposes(similarly to how the facebook app works).

I would also like to try some left/right swiping, similarly to how the facebook app allows you to swipe to bring up the menu or chat.

Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

I have learned many things regarding workflow while desiging apps. One very important thing I've learned is to not put too much time into the design of your app until you have a complete mockup that your happy with. If you spend a lot of time designing before you are finished with a mockup, you might end up having to re-design things as you create more app pages. Or realize you need something that you didn't think of, which could lead to having to redo your layout. It is much more effecient to use minimal time on design until you have all the features you want, and then worry about making it pretty.
